# 2026-03-13

## Release: v1.5.0

Major agent container upgrade + specs housekeeping.

### Rich container tooling

Agent Dockerfile went from minimal (node + chromium + curl + git)
to full workstation:

**Runtimes**: go 1.24, rust/cargo, python3, bun, node
**Package managers**: bun (JS), uv (Python), go install, cargo install
**Media**: ffmpeg, yt-dlp, imagemagick, optipng, jpegoptim
**Research**: pandoc, pdftotext, tesseract-ocr, httrack
**Linters**: biome, ruff, pyright, shellcheck, prettier, htmlhint, svgo
**Data**: pandas, numpy, scipy, matplotlib, plotly, weasyprint
**Office**: marp-cli (slides), python-pptx, openpyxl
**Network**: curl, wget, whois, dig, traceroute
**Search**: ripgrep, fd-find, fzf, tree, bat

WHISPER_BASE_URL now passed to container env — agents can
transcribe audio they download (yt-dlp output, podcasts, etc.)
without going through gateway IPC.

Tools list added to container/CLAUDE.md so agents always see
what's available.

### Channel auth CLI

Generic channel auth flow: `kanipi config <instance> channel auth <name>`.
WhatsApp is the first provider. Interface in `src/channel-auth.ts`,
whatsapp-auth refactored to implement it. Ready for twitter/reddit
auth when those channels ship.
