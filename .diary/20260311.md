# 2026-03-11

## Release: v1.2.0

Shipped three major refactors in v1.2.0:

### Generic Container Commands (J-container-commands spec)

Renamed `runContainerAgent` → `runContainerCommand` with dual execution modes:

- **Agent mode** (default): full ceremony with skill seeding, settings injection, session tracking
- **Raw mode**: skip ceremony, capture stdout as result for bash tasks

Removed `ENTRYPOINT` from Dockerfile — command now supplied via docker args.
This enables scheduler to run arbitrary bash commands in the same container sandbox,
not just full Claude agents.

Added `command` field to `scheduled_tasks` table (migration 0008).

### Media Path Translation (4-paths spec)

Fixed media path translation so agents see container-relative paths:

- Changed `mediaLine()` in `mime.ts` to extract `/media/` portion and prepend `~`
- Agents now see `~/media/20260311/1004/0.png` instead of `/srv/app/home/groups/atlas/media/...`
- Works because `~` = `/home/node/` = mounted group folder in agent containers

### HOST\_\* Path Cleanup (4-paths spec)

Removed redundant `HOST_PROJECT_ROOT_PATH` export:

- Keep three distinct exports: `HOST_GROUPS_DIR`, `HOST_DATA_DIR`, `HOST_APP_DIR`
- Each serves different purpose (groups mount, IPC dir, app source)
- Removed single use of `HOST_PROJECT_ROOT_PATH` in web path construction

### Code Quality

Refine workflow extracted helpers for cleaner organization:

- `appendWithLimit()` — output buffering with limit
- `updateSettings()` — settings.json updates
- `formatMount()` — docker mount formatting
