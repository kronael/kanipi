# 2026-03-12

## Session: IPC cleanup, trigger removal, agent path fixes

### Shipped

**Trigger pattern system removed** — `TRIGGER_PATTERN`, `requires_trigger`,
per-JID auto-prefixing fully deleted. Routing handles everything. Triggers
were redundant since routing was added and caused content mutation (they
auto-prepended `@marinade` to messages). Removed from: `index.ts`, `config.ts`,
`db.ts`, `channels/telegram.ts`, `channels/discord.ts`.

**Errored chat skip** — `isChatErrored()` guard in message loop. Broken
sessions don't retry until user sends a new message.

**`containers wipe` CLI** — `kanipi config <instance> containers wipe` for
manual container force-stop. Removed `cleanupOrphans()` from startup.

**send_file path fix** — `~/` paths now expanded before `/home/node/` strip.
Agents were using `~/tmp/file.txt`, gateway wasn't translating it. Fixed.

**Agent home path instructions** — migration 024: agents must use `~` not
`/home/node/` in all paths, outputs, diary, messages. migration 025: `~/tmp/`
is the standard temp dir; `send_file` requires `~/` paths.

**IDLE_TIMEOUT** — reduced to 60s. Active agents reset the timer on every
output chunk, so only matters for truly hung containers.

**Tier naming** — `minTier` → `maxTier` on actions (tier 0 = most privileged;
old name was semantically inverted).

**Group depth** — `register_group` rejects folders deeper than 3 levels.

**Permissions spec** — fully rewritten: tier model, mount table, gap audit,
delegate/escalate rules, no-parent escalation error.

**Web virtual hosts spec** — simplified: one hostname per world, root-only
routing, vite per world.

**Legacy IPC removed** — `drainLegacyMessages` and `drainLegacyTasks` deleted.
`messages/` and `tasks/` dirs no longer watched. All agent→gateway IPC goes
through `requests/` since v0.5.0; legacy code was dead.

**Tier denial checks** — `ctx.tier === 3` → `ctx.tier >= 3` in delegate_group,
send_file, schedule_task, task management actions.
