# 2026-03-03

## Docs update: workspace layout, skills, discord, character.json

Updated ARCHITECTURE.md, README.md, template/web/pub/howto/index.html
to reflect recent kanipi changes. Also updated takopipi web pages
(intro, kanipi product page) with current architecture.

### Changes documented

**workspace/self mount**: `/workspace/project` (main group only) replaced
by `/workspace/self` (full kanipi source + all groups, read-only). Full
namespace is now: self, group, global, web, ipc, extra.

**/workspace/web rename**: old `/web` mount is now `/workspace/web`.
Web files split into `web/pub/` (public, no auth) and `web/priv/`
(auth-gated). The `/pub/` URL prefix is the auth boundary.

**kronael/tools merge**: development skills (bash, go, python, typescript,
etc.) bundled from kronael/tools into the agent image and seeded to
`~/.claude/skills/` per group on first spawn alongside kanipi skills.

**character.json**: ElizaOS-style agent identity replaces SOUL.md.
Fields (bio, topics, adjectives, style, messageExamples) randomized
per query. Per-instance overrides via `/workspace/global/character.json`.

**migration system**: `container/skills/self/MIGRATION_VERSION` tracks
applied version. `/migrate` skill syncs all groups when version advances.

**signal-driven IPC**: gateway sends SIGUSR1 after writing IPC file.
Agent wakes immediately rather than waiting for 500ms poll interval.

**discord channel**: added via discord.js alongside telegram/whatsapp.
Enabled by `DISCORD_BOT_TOKEN` presence (was already shipped, now documented
properly in ARCHITECTURE.md and README.md).
