# 2026-03-07

## Spec audit, permission model fixes, delegation update

Audited all `specs/1` and `specs/2` files against current code and
wrote `gaps.md` with shipped/partial/open status.

### Docs and spec fixes

- Updated README to stop overstating shipped features
- Rewrote stale specs for auth, voice, worlds, commands, file handling,
  and sidecars to match current implementation
- Rewrote `specs/2/5-permissions.md` to document the actual permission
  model instead of the older aspirational one
- Corrected `specs/1/c-todo.md` status text (`sidecars`, `worlds`, file
  commands, OAuth note)

### Permission model changes shipped

- Top-level non-root folders are now tier 1 worlds
- `main` and `root` are explicit tier-0 aliases
- `register_group` can no longer create new worlds through IPC actions
- Added upward escalation via `escalate_group`

### Delegation rule change

Downward delegation/routing authorization was widened from direct-child only
to descendant-only within the same subtree. Upward escalation remains exactly
one parent hop.

### Validation blocker

Targeted DB-backed tests could not be completed because local
`better-sqlite3` is built for the wrong Node ABI:

- built for `NODE_MODULE_VERSION 137`
- current Node expects `108`
